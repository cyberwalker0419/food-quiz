# 技术栈说明 · 味觉灵魂测试

> 最后更新：2026-06-13

## 项目概览

一个单页 Web 应用，30-60 道趣味测试题，揭示用户最匹配的中国菜系。纯前端、零后端、可生成可分享的卡片图片。

| 项目名 | [food-quiz](food-quiz/) |
| --- | --- |
| 类型 | SPA（单页应用） |
| 部署 | 静态文件（`dist/` 目录可托管到任意 CDN / 静态服务器） |
| 数据 | 100% 内联在 bundle 中，无网络请求 |

### 4. 关键能力

#### 4.1 自适应题目序列（[food-quiz/src/utils/adaptiveQuiz.ts](food-quiz/src/utils/adaptiveQuiz.ts)）

```
入口：pickNextQuestion(pool, answers, usedIds, opts)
├─ aggregateProfile()           // 把已答题目聚合成 8 维口味向量
├─ followUpScore()              // 候选题打分 = 触发器匹配 + 类目覆盖度 + 随机噪声
├─ Early phase: 纯随机        // 前 25% 题目做广度探索
├─ Mid phase:   topK 加权     // 从 top-3 中按 [0.6, 0.3, 0.1] 加权
└─ Final phase: 锁 final 类目  // 终极对决不能漏
```

#### 4.2 匹配算法（[food-quiz/src/App.tsx:42-70](food-quiz/src/App.tsx)）

```
similarity(user, cuisine) = Pearson 中心化余弦相似度
rankCuisines()                // 22 道菜按相似度降序
calculateResults()            // 取 ranked[0] 作为主结果
getSecondaryResults()         // 取 ranked[1..3] 作为副推荐
```

> Pearson 而非欧氏距离：避免"小幅度偏好"被中性 cuisine 拉近（藏菜、新疆菜等 profile 较平）。

#### 4.3 分享卡生成（[food-quiz/src/utils/shareImage.ts](food-quiz/src/utils/shareImage.ts)）

| 步骤 | 实现 |
| --- | --- |
| 1. 等字体加载 | `document.fonts.ready` Promise 包装，避免 CJK 字符绘制失败 |
| 2. Canvas 2D 绘制 | 渐变背景 + 圆角卡片 + emoji + 文字 + 口味条 |
| 3. 导出 PNG | `canvas.toDataURL('image/png')` |
| 4. 下载 | `a.download` 触发浏览器保存 |
| 5. 移动端分享 | `navigator.share({ files: [...] })` 优先，下载兜底 |

#### 4.4 随机种子

- `seed` state 在每次 `startQuiz()` 重新生成
- `pickNextQuestion` 接受 `seed` 选项
- 选题流程已用 `mulberry32` PRNG（见 § 六·当前限制 1），但初始 seed 生成、计算屏文案、PRNG fallback 仍有 4 处 `Math.random()` 残留

## 三、关键设计决策

### 1. 为什么是 React 而不是 Vue / Solid？

- 团队的 React 经验更通用
- 18+ 之后函数组件 + Hooks 已成为事实标准
- Vite 对 React 的一等支持

### 2. 为什么没有用 CSS 框架？

- **品牌一致性**：手写 CSS 让暖色 + 渐变美学有完整控制权
- **首屏体积**：零 CSS 框架依赖，整页 CSS < 30KB
- **学习成本**：新人只需懂 CSS，不用学 Tailwind / styled-components

### 3. 为什么 Pearson 相似度？

欧氏距离在 8 维上对**小幅度偏好**的区分力差：

- 用户选"清淡蒸菜"profile ≈ `{spicy:0, umami:2, sweet:1, sour:1, crunchy:0, tender:2, intense:-3, light:3}`
- 用户选"重辣火锅"profile ≈ `{spicy:3, umami:1, sweet:0, sour:0, crunchy:1, tender:2, intense:3, light:-2}`

Pearson 把"口味形状"而非"绝对数值"做对比，能识别"我也爱清淡"和"我爱辣但不爱酸"这种**方向性**的相似。

### 4. 为什么题目 151 道 / 显示 60 道？

- **多样性**：完整版 60 题从 151 道中选，覆盖 10 个 category（daily / mood / texture / adventure / ingredient / sensory / social / culture / culture-deep / final）
- **快慢可选**：精简版从 `version: 'quick'` 标记的子集中抽 30 道
- **重玩价值**：自适应算法基于 seed 重排，两次跑完 60 道题几乎不重复

### 5. 为什么没有 i18n？

- 第一版本专注中文用户
- 文字数量 < 200 条（题面 + 菜名 + 描述 + 性格标签）
- 未来若国际化：所有字符串可集中到 `i18n/zh-CN.ts` / `en-US.ts`

### 6. 为什么没有后端 / API？

- **隐私**：用户的口味数据只在本地，不上传
- **零成本**：纯静态托管，Netlify / Vercel / GitHub Pages 均可
- **离线可用**：未来可加 PWA / Service Worker

## 六、已知限制 / 改进方向

### 当前限制

1. **`seed` 未完全生效**：`mulberry32` PRNG 已实现（[adaptiveQuiz.ts](food-quiz/src/utils/adaptiveQuiz.ts) `mulberry32()`、[App.tsx:116](food-quiz/src/App.tsx) 选题流程已用），但仍有 4 处 `Math.random()` 残留 —— 初始 seed 生成（`App.tsx:104, 140`）、计算屏文案随机选（`App.tsx:324`）、PRNG fallback（`adaptiveQuiz.ts:36`）。
2. **藏菜 / 徽菜 / 新疆菜等平坦 profile 偶尔被推为第一**（Pearson 特性）
3. **`transition: all` 仍出现 1 处**（[App.css:27](food-quiz/src/styles/App.css) 全局变量定义，违反 Web Interface Guidelines）
4. **缺 `prefers-reduced-motion` 媒体查询**：动画对前庭敏感用户不友好
5. **缺 `aria-live` / `aria-hidden` 标注**：copy-toast 不会朗读，浮动 emoji 会被读屏念
6. **`index.html` 仍有 `user-scalable=no`**：违反无障碍最佳实践
7. **微信 / 微博 / 小红书分享按钮未实现**（仅有 Web Share API + 下载）

### 后续 TODO
